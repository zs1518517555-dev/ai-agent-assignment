# 深度学习与生命科学 —— AI Agent 课程项目

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

> "深度学习与生命科学"课程项目作业，聚焦于构建面向蛋白冠研究的 AI Agent 知识问答平台。

## 📋 项目概述

本仓库包含课程项目的完整作业规范。学生将使用多智能体协作技术构建一个智能问答系统。项目分为**必修基础模块**和**三级难度的自选进阶模块**。

**总分：100 分**
- 知识资产（35 分）
- 评测基准（35 分）
- 知识问答项目（30 分）

## 🎯 基础模块（必修）

所有学生必须完成以下三个模块：

### 1. 知识资产（35 分）
收集并整理至少 20 个与计算生物学和深度学习相关的知识集。每个知识集包含：
- `content/` —— 原始内容（论文、网页等）
- `keywords.json` —— 关键词与主题分类
- `source.json` —— 来源元数据（DOI、URL 等）

**研究方向主题**：学生可从以下主题中选择一个作为研究方向：

| 主题 | 说明 |
|------|------|
| `protein-corona` | 蛋白冠 |
| `single-cell-omics` | 单细胞组学 |
| `genomics-sequencing` | 基因组学与测序 |
| `gene-perturbation` | 基因扰动 |
| `drug-discovery` | 药物发现 |
| `protein-structure` | 蛋白质结构 |
| `neural-network` | 神经网络 |
| `graph-neural-network` | 图神经网络 |
| `generative-model` | 生成模型 |
| `foundation-model` | 基础模型 |
| `multi-omics` | 多组学整合 |
| `spatial-biology` | 空间生物学 |
| `epigenomics` | 表观基因组学 |
| `biomedical-nlp` | 生物医学自然语言处理 |
| `molecular-dynamics` | 分子动力学 |
| `medical-image-reconstruction` | 医学图像重建 |
| `medical-image-segmentation` | 医学图像分割 |
| `medical-image-classification` | 医学图像分类/识别 |
| `medical-image-registration` | 医学图像配准 |
| `medical-image-synthesis` | 医学图像合成 |

学生可以自定义关键词（使用 kebab-case 格式），但主题应与所选研究方向一致。

### 2. 评测基准（35 分）
构建覆盖知识库的测试集，至少 20 道题目。每道题包含：
- 题目文本与标准答案
- 来源引用
- 主题分类
- 难度评级

### 3. 知识问答项目（30 分）
通过选择自选模块实现智能问答系统。

**模块选择规则**：所选模块的难度值之和必须 **≥ 4**。

有效组合示例：
- 一个 Lv.2 + 一个 Lv.2（2 + 2 = 4）
- 一个 Lv.1 + 一个 Lv.3（1 + 3 = 4）
- 两个 Lv.1 + 一个 Lv.2（1 + 1 + 2 = 4）
- 一个 Lv.1 + 两个 Lv.2 + 一个 Lv.3（1 + 2 + 2 + 3 = 8）—— 超过最低要求，允许

最终得分计算方式：
```
加权总分 = Σ(模块得分 × 模块权重)
知识问答项目得分 = 加权总分 × 0.3
```

## 🚀 自选模块（自选）

学生从以下三类任务中选择，每个模块有难度等级（1=基础，2=中等，3=挑战）。**所选模块的难度值之和必须至少为 4。**

### 🛠️ 技能系统（`技能.md`）
| 模块 | 等级 | 描述 |
|--------|-------|-------------|
| 部署 MCP 工具/Skill | Lv.1 | 将现有 MCP 工具接入课程平台 |
| 构建领域知名项目的 MCP/Skill | Lv.2 | 为生物信息学工具（UniProt、PDB、BLAST 等）创建 MCP 接口 |
| 大规模技能路由系统 | Lv.2 | 实现支持 ≥2000 个技能的语义检索路由 |
| 自进化技能系统 | Lv.2 | 从使用日志中提取模式，自动生成新技能 |

### 🔀 工作流编排（`编排.md`）
| 模块 | 等级 | 描述 |
|--------|-------|-------------|
| 基础静态编排 | Lv.1 | 固定多智能体协作工作流（≥4 个智能体） |
| 基础动态编排 | Lv.1 | 动态创建智能体并并行执行 |
| 进阶静态编排 | Lv.2 | 预定义工作流模板，支持 JSON/YAML 配置 |
| 进阶动态编排 | Lv.3 | 自动生成工作流，支持规划与错误恢复 |

### 🧠 记忆系统（`记忆.md`）
| 模块 | 等级 | 描述 |
|--------|-------|-------------|
| 扩充知识资产 | Lv.1 | 新增 ≥30 个知识集，附覆盖度分析 |
| 结构化知识库 | Lv.1 | 构建实体关系图用于知识检索 |
| Agentic RAG | Lv.1 | 实现带自纠正的迭代检索 |
| 持久化记忆系统 | Lv.1 | 多层记忆架构（工作记忆/情景记忆/语义记忆） |
| 多跳推理 | Lv.2 | 复杂多步检索，带验证循环 |
| 多模态知识库 | Lv.2 | 支持文本/图像/表格多模态 |
| 自进化记忆 | Lv.3 | 自动提取经验单元，优化记忆结构 |

## 📊 评分标准

详见 [评分标准](ch/评分标准.md) 或 [grading_criteria](en/grading_criteria.md)。

**评分原则：**
- 基础模块侧重完整性与质量
- 自选模块侧重功能实现与可演示结果
- 难度等级越高的模块在最终得分中权重越大
- 代码必须可运行，文档必须完整

## 🛠️ 推荐技术栈

- **智能体框架**：Agno、LangChain、LangGraph、CrewAI 或 AutoGen
- **知识库**：向量数据库（Chroma、Weaviate、Milvus）或图数据库（Neo4j）
- **MCP 协议**：用于工具集成与技能标准化
- **评测**：自定义评测套件，支持自动评分
